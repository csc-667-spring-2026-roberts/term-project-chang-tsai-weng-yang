About This Project

886 Gin Rummy is a two-player card game project built for CSC 667. We focused on clear gameplay flow, authentication, and real-time updates with Server-Sent Events.

What We Built

  • Playable Gin Rummy game interface with draw, discard, and knock actions.
  • User authentication with register, login, logout, and protected route checks.
  • SSE pipeline for server-to-client updates with subscription and broadcast support.
  • Automated tests for SSE endpoint behavior and multi-client event delivery.

Design Goals

  • Simple UX: Keep game controls readable and easy to follow.
  • Reliable state: Push updates from server instead of polling.
  • Consistent styling: Shared nav, color system, and card layout.
  • Maintainable code: TypeScript structure with focused modules.
